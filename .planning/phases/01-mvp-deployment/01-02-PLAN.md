Human tests each module on the PRODUCTION URL. Use a desktop browser first, then repeat critical flows on a mobile viewport (Chrome DevTools responsive mode or actual phone).

QA Checklist:

1. Authentication (AUTH-01 through AUTH-04):
   - Navigate to production URL — should redirect to /login
   - Enter invalid credentials — should show error message
   - Enter valid credentials — should redirect to dashboard
   - Refresh page — should stay on dashboard (session persists)
   - Click logout — should return to /login
   - Try accessing dashboard URL directly when logged out — should redirect to /login

2. Dashboard (DASH-01 through DASH-05):
   - 4 metric cards visible: Balance, Ingresos, Egresos/Gastos, Inversiones
   - Period selector works: navigate to previous/next month
   - Metrics update when changing period
   - Last 5 movements section shows recent entries (or empty state if none)
   - Quick action buttons present for each movement type
   - 6-month bar chart renders (income vs expenses)

3. Movement Registration (MOV-01 through MOV-07):
   - Click quick action button — modal opens with correct type pre-selected
   - Form fields present: description, amount, date, category, notes
   - Form color adapts to selected type (green=INGRESO, red=EGRESO/GASTO, amber=INVERSION)
   - Category dropdown filters by selected type
   - Submit a test INGRESO — appears in dashboard last 5 and history
   - Edit the movement inline — changes persist
   - Delete the movement — disappears from list (soft delete)

4. History (HIST-01 through HIST-06):
   - Navigate to history page — all active movements visible
   - Movements grouped by date
   - Each row shows: ID (JY-XXXX), description, category, amount with type color
   - Filter by type — only that type shown
   - Filter by category — only that category shown
   - Filter by date range — correct movements shown
   - Period summary shows income, expenses, balance totals
   - Search by description — filters correctly

5. Type Views (VIEW-01 through VIEW-03):
   - /ingresos page — shows only INGRESO movements
   - /egresos page — shows only EGRESO and GASTO movements
   - /inversiones page — shows only INVERSION movements

6. Settings (CONF-01 through CONF-03):
   - Categories list visible with seed data
   - Create a new category — appears in list
   - Edit category name inline — change persists
   - Deactivate a category — visual indicator changes
   - Reactivate the category — returns to active state
   - User profile section visible (read-only)

7. Mobile Responsiveness:
   - Dashboard cards stack on mobile (not side-by-side overflow)
   - Sidebar collapses or is accessible via hamburger menu
   - Movement form modal is usable on mobile
   - History page is scrollable and readable on mobile
   - All text is legible, no horizontal overflow

Report any failures with: module name, expected behavior, actual behavior.
